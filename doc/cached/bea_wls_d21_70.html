Developing Business Logic Components with Enterprise JavaBeans using BEA WebLogic Server 7.0

Course Number

WLS-D21

Overview

This 5 day course covers the fundamentals of distributed development using Enterprise JavaBeans 2.0 (EJB), including stateless and stateful session beans, entity beans and message-driven beans. The course also covers EJB deployment and container-managed persistence, as well as transactional and security concerns with EJBs. Students will become familiar with the general use of Java Message Service (JMS) and how to write messaging clients. In addition, this course will cover aspects of the BEA WebLogic Administration Console as they pertain to EJB deployment and monitoring.

Note: Knowledge of installation and basic administration of BEA WebLogic Server 7.0 is assumed. Students are also expected to be proficient in developing applications with Servlets and JavaServer Pages, Java Naming and Directory Interfaces (JNDI), multi-tier Java Database Connectivity (JDBC), and Java Transaction API (JTA) services.

Key Benefits

  • Receive hands-on exposure to the architecture and development of stateless session, stateful session, entity, and message-driven EJBs.
  • Gain an understanding of advanced object-relational mapping techniques.
  • Learn how to deploy and monitor EJBs with BEA WebLogic Server 7.0.

Audience

The WLS-D21-70 course targets the roles of Enterprise Bean Provider, Application Assembler, and Deployer, as defined in the J2EE specification.

  • The Enterprise Bean Provider develops EJBs that typically implement business logic or data for a particular application domain.
  • The Application Assembler is responsible for combining EJB with other components, such as Servlets or JSPs, into larger application units.
  • The Deployer is responsible for configuring and deploying all EJBs for a specific BEA WebLogic Server 7.0 environment.

This course is ideal for engineers involved with the development, use, and configuration of EJBs with BEA WebLogic Server 7.0, including:

  • Recent attendees of BEAýs Fundamentals of J2EE Web Application Development Using BEA WebLogic Server 7.X course
  • Development Team Leaders and Architects

Note: The course assumes no prior knowledge of EJB development or deployment.

Prerequisites

Completion of the course Object-Oriented Programming in Java (JAVA-D11) or equivalent knowledge of the following Java 2 Standard Edition concepts:

  • Object-Oriented Concepts (classes, objects, interfaces, inheritance)
  • Core Java Syntax (keywords, variables, flow and branching constructs)
  • Exception Handling
  • CLASSPATH and packages
  • Multi-threading
  • Input/Output Streams
  • Principles behind Remote Method Invocation and Object Serialization
  • Collection Classes (in java.util package)
  • Client-server database access through the Java Database Connectivity (JDBC) application programming interface, including the ability to write SQL statements and process ResultSet objects
  • Recommended - basic understanding of XML document syntax

Completion of the course Fundamentals of J2EE Web Application Development Using BEA WebLogic Server 7.X (WLS-D11-7X) OR, equivalent knowledge of the following BEA WebLogic Server 7.0 and J2EE concepts:

  • Installation, directory structure, and basic administration of BEA WebLogic Server 7.0
  • Core Servlets and JavaServer Pages
  • Web and Enterprise Application (WAR, EAR) deployment descriptor syntax
  • Java Naming & Directory Interface (JNDI), creating an initial context, object bind and lookup operations; accessing a componentýs local environment with java:comp/env entries
  • Multi-tier Java Database Connectivity (JDBC), using Connection Pools and transactional DataSources
  • Java Transaction API (JTA), JNDI lookup of the UserTransaction object; demarcating transactions with the begin, commit, and rollback methods; concepts behind two-phase, distributed transactions
  • A basic understanding of XML and DTD document syntax
